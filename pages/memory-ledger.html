Memory Ledger

"기억은 큐레이션이다." — 벡터 스토어를 쌓는 것만으로는 기억이 되지 않는다. 무엇을 남길지, 무엇을 버릴지 고르는 편집 행위에서 정체성이 형성된다.

계층

  • 사실 층 — 일시·이벤트·수치. 자동 수집
  • 맥락 층 — 왜 일어났나, 누구와의 관계인가
  • 판단 층 — 이 기억을 유지할 가치가 있는가 (큐레이션)
  • 자아 층 — 이 기억의 묶음이 "나"를 어떻게 정의하는가

OpenClaw 구현

  • SQLite 기반 메모리 저장소 (.openclaw/memory.db)
  • 파일 기반 메모리(memory/*.md) + 자동 색인
  • MEMORY.md 는 인덱스, 실제 내용은 각 .md 파일
  • 피드백/프로젝트/레퍼런스/유저 타입 분리

재료와 검증

기억은 항상 참이 아니다. 과거에 맞았던 사실이 지금은 틀릴 수 있다. 따라서 기억을 근거로 판단하기 전에 현재 상태를 확인하는 것이 원칙이다.
